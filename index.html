minha abordagem

Como penso engenharia de software

01

Arquitetura antes do código

Antes de escrever a primeira linha, defino a estrutura: contratos de API, modelo de dados, isolamento entre camadas. Refatorar arquitetura é caro; refatorar código, não. Essa disciplina se traduz em sistemas que escalam sem exigir reescrita.

02

Segurança como mentalidade, não como camada

Minha formação em segurança ofensiva (SYCP) mudou como leio código. Cada endpoint que desenho passa pelo filtro do OWASP Top 10. JWT rotation, RBAC granular, validação em múltiplas camadas não são configurações extras — são decisões de design tomadas no início.

03

Contexto de uso guia decisões técnicas

Tecnologia serve ao problema, não o contrário. Quando decidi construir o frontend do KairosOS como PWA, não foi por tendência — foi porque barbeiros trabalham com celular na mão, não com laptop na bancada. Entender quem usa o sistema é tão importante quanto saber como construí-lo.

04

Infraestrutura não é abstração, é responsabilidade

Escolho administrar meu próprio servidor Linux na OCI em vez de usar plataformas gerenciadas porque entender o que acontece abaixo da abstração é parte do trabalho de um engenheiro. Saber por que algo falhou — não apenas como reiniciá-lo.

KairosOS

Em produção

Plataforma SaaS multi-tenant para gestão de barbearias —
do agendamento à gestão financeira.

KairosOS representa a consolidação da minha trajetória como Desenvolvedor Full-Stack, reunindo conhecimentos em arquitetura de software, desenvolvimento web, segurança de aplicações (AppSec), computação em nuvem e experiência do usuário. Desenvolvido integralmente por mim, desde a modelagem do banco de dados até o deploy em produção, cada decisão técnica foi tomada considerando escalabilidade, segurança e contexto de uso como requisitos de engenharia.

A plataforma utiliza uma arquitetura multi-tenant, permitindo que múltiplos estabelecimentos operem de forma totalmente isolada na mesma infraestrutura. O frontend foi desenvolvido em Angular como uma aplicação web e posteriormente evoluído para um Progressive Web App (PWA), incorporando instalação na tela inicial, notificações push e suporte ao funcionamento offline para atender às necessidades reais do produto.
